# 既存サイトATDD適合度レビュー結果 (existing_site_atdd_review.md)

## 📊 適合評価結果
- **適合スコア**: 25% / 100%
- **適合ステータス**: 改善要求あり (不適合)

## 🤖 定期レビューアーからの指摘内容

既存の予約サイトの仕様と、動的クローン（上田様、田中さん、小林さん）の葛藤を解消するために策定したATDD基準（Given-When-Then 10シナリオ）を対比し、既存サイトが受入基準にどれだけ適合しているかを厳格にレビューしました。

■ 判定結果: **不適合 (適合度: 25%)**

■ 不適合（ギャップ）の詳細:

1.  **ビジネス出張者向け要件 (US-01 〜 US-03) の乖離 (未実装率: 100%)**:
    - **宿泊日指定の空室プランダイレクト表示 (US-01)**: 宿泊日を指定した検索機能（`#checkin-date`、`#search-vacant-btn`）がなく、満室プランが強制表示され、かつプラン選択時のリアルタイム在庫チェックAPIが実装されていません。
    - **旅費規程上限1万円ロック (US-02)**: 出張モードのチェックボックス（`#limit-lock-checkbox`）および10,001円以上のプランの選択ボタンを自動的に `disabled` にして警告バッジを出すガードUIがありません。
    - **会社宛電子領収書の事前指定 (US-03)**: コーポレート決済選択時に宛名「プレインリビング株式会社」を指定して、予約完了と同時にPDF領収書を自動発行・ダウンロードする処理が存在しません。
2.  **PC出張者向け要件 (US-04 〜 US-05) の乖離 (未実装率: 100%)**:
    - **ドラフト暗号化自動保存 (US-04)**: 入力途中のデータをAES暗号化してLocalStorageに保存し、2時間で自動消去するライフサイクル管理、および再開時の復元ダイアログ（`#restore-draft-dialog`）がありません。
    - **共有PC自動保存無効化 (US-05)**: 共用PCチェックボックス（`#shared-pc-checkbox`）のON時に、LocalStorageへの保存処理を完全にブロックするバイパス制御がありません。
3.  **シニア代理予約者向け要件 (US-06 〜 US-10) の乖離 (未実装率: 100%)**:
    - **WebOTPによる1タップ認証コード自動入力 (US-06)**: SMSで届いたコードを `navigator.credentials.get` 経由で1タップ自動転記するWebOTP連携処理がありません。
    - **A4しおり印刷専用CSS (US-07)**: `@media print` 印刷時にヘッダーやナビゲーションを非表示にし、しおり情報を極大フォント（30px以上）でA4用紙1枚にレイアウトする印刷CSSが定義されていません。
    - **2秒長押し確定ボタン (US-08)**: 指の震えやスクロール時の誤タップを防ぐための、2秒ホールドでゲージが溜まる決済確定ボタン（`#hold-confirm-btn`、`aria-valuenow`）がありません。
    - **厨房アレルギー同期 & ハートビート監視 (US-09)**: 厨房DBへのアレルギー情報の自動同期および毎分の監視処理がなく、同期エラー時に点滅警告（`#allergy-sync-alert`）をフロント画面に出す安全保護設計がありません。
    - **電話サポートセッション引き継ぎ (US-10)**: 4桁の一時共有コード（`#transfer-code-display`）を発行し、コールセンターのスタッフ画面に入力途中データを瞬時にロードする連携APIがありません。

■ 改善推奨アクション:
既存予約サイトは、動的ユーザーが抱える「経理の叱責への恐怖」「データ消失のイライラ」「操作タイムアウトのパニック」「アレルギー伝達漏れによる生命危機への不安」という精神的ペインに一切対応できていません。上記の10個のギャップを解消するため、フロントエンド（HTML/CSS/JS）、バックエンド（FastAPI/同期API/一時セッションキャッシュ）の全面的な改修が必要です。
