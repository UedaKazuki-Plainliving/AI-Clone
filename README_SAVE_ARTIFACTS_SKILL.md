# 成果物世代管理スキル（save_artifacts_timestamped）設置手順書

本手順書は、今回のシミュレーションで構築した「成果物タイムスタンプ世代管理スキル」を社内のAntigravity環境へ配置するための手順と、配置パスをまとめたものです。

---

## 1. ファイルの配置場所（絶対パス）

社内環境のAntigravityワークスペース（例：`C:\Users\plain\antigravity`）に対して、以下の通りファイルを配置してください。

### ① スキル定義ファイルの配置
*   **配置先パス**: 
    `C:\Users\plain\antigravity\.agents\skills\save_artifacts_timestamped\SKILL.md`
*   **ファイル内容**:
    下記「2. スキル定義ファイル（SKILL.md）」の内容をそのままテキストファイルとして新規作成・配置してください。

### ② 格納庫（アーカイブ用フォルダ）の作成
*   **配置先パス**:
    `C:\Users\plain\antigravity\archives\`
*   **手順**: 
    `archives` という名前のフォルダをワークスペース直下に作成してください。実行時に自動的にこの中にタイムスタンプ付きフォルダが作成され、成果物が保存されます。

---

## 2. スキル定義ファイル（SKILL.md）

以下が、配置するスキルの内容です。

```markdown
---
name: save_artifacts_timestamped
description: 現在のシミュレーション成果物（simulation_results/ 配下の全ファイル）を、実行時の日時タイムスタンプ（YYYYMMDDhhmmss形式）のアーカイブディレクトリにコピーして格納し、世代管理用の格納庫（archives/）を構築・更新する。
---

# Skill: save_artifacts_timestamped (成果物タイムスタンプ世代管理スキル)

## 1. 目的

本スキルは、実行時に生成されたATDDシミュレーション結果（`simulation_results/`）を、実行時の日時スタンプ（`YYYYMMDDhhmmss`）に基づいた専用のアーカイブディレクトリに自動で格納し、過去の要件定義・テスト仕様の変遷を失わずに世代管理（格納庫の構築）を行うためのガイドラインです。

---

## 2. 進行プロセス

1.  **タイムスタンプの取得**:
    - 実行時のローカル日時（例: `2026-06-22 07:35:58`）を `YYYYMMDDhhmmss` 形式（例: `20260622073558`）にフォーマットします。
2.  **格納庫ディレクトリの作成**:
    - ワークスペースのルートに `archives/` ディレクトリを作成し、その配下にタイムスタンプ付きのフォルダ（例: `archives/20260622073558/`）を作成します。
3.  **成果物のコピー**:
    - `simulation_results/` ディレクトリ配下にあるすべてのファイル（`user_clones.md`、`interview_log_autonomous.md`、`playwright_test.spec.js` 等）を、作成したタイムスタンプ付きフォルダにそのままコピーします。
4.  **格納完了の報告**:
    - 格納されたフォルダの絶対パスと、格納されたファイル一覧をユーザーにわかりやすく報告します。
```

---

## 3. 社内環境への導入手順

1.  上記「1.」に記載されたパスにフォルダおよびファイルを作成します。
2.  次回以降、Antigravityでシミュレーションを実行した直後、または成果物を整理したい時に、**「スキル save_artifacts_timestamped を発動して」** と指示することで、本スキルが実行され自動で成果物がフォルダ分けして格納されます。
