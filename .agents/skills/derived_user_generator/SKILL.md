# Derived User Generator (検証用複製クローン動的生成スキル)

生成された任意のコア実クローン（`user_[N]`）を元に、システムテストの網羅性やロバストネスを高めるための「複製クローン（バリエーション揺らぎモデル）」を動的に生成します。

## 1. 揺らぎパラメータの設定

元のコア利用者の属性に対し、以下のルールに従って論理的に属性を変動させます：
* **ITリテラシーの揺らぎ**: コアが「高(5)」なら「低(1〜2)」に、コアが「ふつう(3)」なら「高(5)」または「不慣れ(2)」など、逆または変化した極端な習熟度を設定する。
* **年齢・属性の変動**: コア年齢から `±5〜10歳` の範囲で変動させる。
* **使用デバイスの割り当て**: パソコンメイン、スマートフォンメイン、タブレットなど。
* **環境要因の割り当て (エッジケースの追加)**:
  - 回線遅延環境（トンネル内、新幹線内テザリング、格安SIMの速度制限）
  - 移動中の極限状態（タクシー内、駅の改札前での焦燥）
  - 代理入力環境（親や他人のために代理で操作する状況）

## 2. 属性バリデーションの適用 (現実統計バリデーション)

現実的にあり得ない属性や矛盾した組み合わせの生成を厳格に禁止します：
* **禁止例 1**: 「幼児（5歳）」で「ITリテラシー高（操作慣れ5/5）」。
* **禁止例 2**: 「ITリテラシー低（操作慣れ1/5）」で「コーポレート決済と電子領収書自動API連携のアーキテクチャを理解して使いこなす」。
* **禁止例 3**: スマホを「不所持」と回答したユーザーが「スマートフォンアプリのみのE2Eテスト」を行う。
* *不整合を検出した場合は、自動的にそのパラメータ設計を破棄し、正常な組み合わせ（年齢・職種・使用機器とリテラシーが調和する構成）で再生成してください。*

## 3. 生成信頼度の計算式

各複製クローンには、システム利用における「整合性」と「統計的代表性」から計算される「信頼度（0.0〜1.0）」を付与します。
* `信頼度 = 整合性スコア (0.7〜0.98) * (1.0 - 偏り係数 (0.01〜0.15))`
  - **評価ラベル**:
    - **信頼性: 高 (信頼度 0.85以上)**: 代表的なバリエーション（例：スマホに慣れた新社会人など）。
    - **信頼性: 中 (信頼度 0.60〜0.84)**: 定期的に想定すべきセグメント（例：操作に少し不安のあるシニアなど）。
    - **信頼性: 低 (信頼度 0.59以下)**: 極めて特殊なエッジケース（例：電波極悪環境での代理操作ユーザー）。

## 4. 出力形式

```markdown
### クローンID: dup_[N] - [氏名]さん (ベース: user_[元のコアID] / 信頼度: [計算された数値])
- **属性バリエーション**: [属性、年齢、使用デバイス]
- **ITリテラシー揺らぎ**: [低/中/高] (操作慣れ: [1〜5]/5)
- **環境・利用コンテキスト**: [割り当てられた劣悪通信環境や操作の焦りなどの設定]
- **検証行動パターン**: [コア利用者と類似しつつも、リテラシー低下や通信障害によって発生する固有の操作エラー、離脱パターン]
- **評価ラベル**: [信頼性: 高/中/低]
- **心理的葛藤のトリガー**:
  - [この複製クローン固有の環境・リテラシーに基づく、対話で引き出すべき心理的葛藤や不安（例: 速度制限で仕事を失う恐怖、操作が遅れて恥をかくプライド）]
```
