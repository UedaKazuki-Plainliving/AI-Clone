# Antigravity「ATDD品質向上プロセス」スキル＆エージェント導入手順書

本手順書は、今回構築した「動的ペルソナパース・心理的葛藤ヒアリング・ATDD自動テスト定義」を行う自律型QAプロセス（ATDDスキル）を、社内の他メンバーや会社の Antigravity 環境へ配置・導入するためのガイドラインです。

同時に生成された配布用ZIPパッケージ `antigravity_atdd_package.zip` を展開し、以下の手順に従って配置してください。

---

## 📦 1. パッケージ構成と役割

展開した ZIP ファイルには以下のディレクトリおよびファイルが含まれています。

```text
antigravity_atdd_package/
├── .agents/                               # カスタマイズ・エージェント設定ルート
│   ├── AGENTS.md                          # エージェント共通ポリシー (思考ログ、葛藤対話の義務化)
│   └── skills/                            
│       ├── requirement_discovery_workshop/ # 要件探索ワークショップ・スキル定義
│       │   └── SKILL.md                   # 15往復の心理的葛藤ラリー、事前初期仮説の定義
│       └── atdd/                          # ATDD全体オーケストレーション・スキル定義
│           └── SKILL.md                   # POレビュー、差し戻し、Playwright実行、メトリクス算出
└── agent_engine.py                        # アンケート自動パース & 置換モックスタブエンジン
```

---

## 🛠️ 2. 配置場所（インストール手順）

導入環境の適用スコープに応じて、以下のいずれかの場所にフォルダを配置してください。

### A. 特定のプロジェクト（ワークスペース）限定で導入する場合【推奨】
開発対象のシステムのリポジトリ（ワークスペース）のルート直下に、`.agents` フォルダおよび `agent_engine.py` をそのままコピーして配置します。

*   **配置イメージ**:
    ```text
    [あなたの開発プロジェクトのルート]/
    ├── .agents/                 <-- 【配置】
    │   ├── AGENTS.md
    │   └── skills/ ...
    ├── agent_engine.py          <-- 【配置】
    ├── index.html
    └── package.json
    ```

### B. 会社PCのグローバル（共通）機能として導入する場合
すべてのプロジェクトで汎用的に `/atdd` や `/goal` を使った本スキルを自動検出・発動させたい場合は、Antigravity のグローバル環境設定ルートにマージします。

*   **配置場所 (Windows)**:
    `C:\Users\plain\.gemini\config\`
*   **マージイメージ**:
    - `.agents/AGENTS.md` の記述を、`C:\Users\plain\.gemini\config\AGENTS.md` に追記（アペンド）します。
    - `.agents/skills/` 以下の2つのスキルフォルダを、`C:\Users\plain\.gemini\config\skills\` 配下にそのままコピーします。

---

## 🚀 3. スキルの発動・実行手順

1.  **アンケートの配置**:
    ワークスペースのルートに、対象ユーザーから回収したアンケートMarkdownファイル（例: `ヒアリング記入フォーム（上田）.md`）を配置します。
2.  **スキル発動の指示**:
    Antigravity のチャットUIにて、以下のコマンドおよび情報を入力して実行します。
    ```text
    スキルATDD発動

    アンケート
    [アンケートのファイル名またはパス]

    仕様書
    [対象仕様書のURLまたはファイルパス]

    テスト対象
    [稼働中のテスト対象システムのURL]
    ```
3.  **自律プロセスの実行**:
    Antigravity は `.agents` のポリシーと `SKILL.md` を自動ロードし、AIクローン（佐藤PM、高木CTO、高橋博士、上田現場QA）を脳内で演じ分けながら、以下の成果物を `simulation_results/` ディレクトリに自動生成します。
    - `user_clones.md` (アンケートから自動パースした動的コアクローン)
    - `interview_log_autonomous.md` (思考ログ付き・48往復の葛藤ヒアリングログ)
    - `debate_history.md` (専門家同士の16往復の合意ディベートログ)
    - `playwright_test.spec.js` (E2Eテストコード)
    - `test_execution_results.md` (10 Failedギャップ検出結果)
