# Skill: Expert Reviewer (Multi-Agent Document Reviewer)

## 1. Purpose
本スキルは、開発中の「要件定義書」「基本設計・仕様書」「テスト設計書」などの各種設計ドキュメントをインプットとして読み込み、品質向上マルチエージェント（QA工学博士・高橋、CTO・高木、現場ベテラン・上田、PO・佐藤）がそれぞれの専門観点からレビューを実行し、改善案と品質の評価判定を含む詳細なレビューレポート（`review_report.md`）を出力するための手順書です。

本スキルにより、自動テストスクリプトを構築・実行する前段階、あるいはドキュメント作成の初期フェーズにおいて、仕様の曖昧さ、テスト容易性、利用時品質リスク、ビジネス目標との乖離、技術制約を早期に検出（静的テスト / レビューの自動化）します。

---

## 2. 進行プロセス

本スキルを実行するエージェントは、以下のステップを自律的に進めます。

### Step 1: ドキュメント種別と基本情報の把握
* **アクション**: 入力されたドキュメントの内容を解析し、以下のいずれの種別であるかを特定します。
  1. **要件定義書**: システムが満たすべき要求、機能、非機能要件、ユースケースなどが記載されたファイル。
  2. **基本設計・仕様書**: 画面遷移、データ入出力、バリデーションロジック、データベース設計、API仕様などが記載されたファイル。
  3. **テスト設計書**: テスト観点、テスト条件、テストケース、期待値などが記載されたファイル。
* **出力バッファ**: 特定したドキュメント種別、プロジェクト名、対象機能の概要。

### Step 2: 各専門クローンによる個別レビュー実行
`.agents/AGENTS.md` に定義されている各専門クローンのレビュー評価観点に基づいて、ドキュメントの記述内容を厳密にチェックします。

1. **高橋寿一の品質工学的指摘 (QAエキスパート)**:
   - *要件・仕様・テスト設計*: 曖昧さの有無、テスト容易性、同値分割・境界値分析、網羅性。インタビューを通じて要求・ユーザーストーリーが適切に引き出されているかの検証。
2. **高木洋平のアーキテクチャ・ガバナンス指摘 (CTO/QAエキスパート)**:
   - *要件定義/仕様書*: ビジネス目標との整合、非機能要件、セキュリティ基準。インタビューアーとしてシステム運用・ガバナンスの視点から要求を引き出す。
3. **上田和樹の現場・利用時品質指摘 (現場QAエキスパート)**:
   - *要件・仕様・テスト設計*: 利用時品質リスク、エラーハンドリング、レスポンス・待機設計、アクセシビリティ、現場ユーザーシナリオの妥当性検証。
4. **ATDDエンジニアの自動化フィジビリティ指摘 (ATDDエンジニア)**:
   - *自動化実現性*: Gherkin受入基準の作成と、5大技術観点（ロケーターの特定容易性、操作手順の再現可能性、期待結果のアサーション可能性、前提条件の確立容易性、テストデータの確保・制御性）に基づくテストコード実装可能性（自動化が回るか）の検証・レビュー。
5. **プロダクト責任者（佐藤）の品質・プロジェクト継続実現性指摘 (Project Manager / Product Owner)**:
   - *ATDD品質寄与度*: 予防品質、機能適合性、デグレ防止等への寄与（有効性）。
   - *インタビュー有効性*: 心理的葛藤ラリーにより本質的な利用時品質の観点・価値を引き出せているか。
   - *プロジェクト継続実現性*: スケジュール、工数、QAエンジニアリソース、インフラコスト、自動テストのメンテナンス負荷等を含めたプロジェクト管理面での継続運用性。
   - *ビジネス価値*: 顧客価値、MVPとしての適切さ。

## 3. レビュー実行ルール
* **推測と事実の分離**: レビュー指摘は感覚的ではなく、「〇〇の記載がないため、〇〇のテストが設計できません」といった論理的かつ具体的な根拠を明記すること。
* **日本語での出力徹底**: 指示、成果物、対話、思考プロセス、報告書を含め、すべてのエージェント出力およびドキュメント生成は日本語（Japanese）で行うこと。
